How It Works

Three simple steps to add memory to your voice agents

1

Deploy in Minutes

Run docker-compose up -d to start the entire stack. Configure your .env file with API keys. Point ElevenLabs webhooks to your endpoint.

git clone https://github.com/webmasterarbez/elaaoms_claude.git
cd elaaoms_claude
cp .env.example .env
docker-compose up -d
2

Automatic Processing

When calls end, ElevenLabs sends transcription webhooks. ELAAOMS extracts memories using LLM, deduplicates against existing data, and stores in OpenMemory.

  • Pre-call personalized greetings
  • During-call memory search
  • Post-call memory extraction

Frequently Asked Questions

Everything you need to know about ELAAOMS

What is ELAAOMS?

ELAAOMS (ElevenLabs Universal Agentic Open Memory System) is an open-source memory system that automatically extracts, stores, and retrieves conversation memories for ElevenLabs voice AI agents. It provides personalized greetings for returning callers and real-time memory search during calls.

How much does it cost?

ELAAOMS is completely free and open source (MIT license). The only costs are your infrastructure (self-hosted or cloud) and LLM API costs (~$0.026 per call with OpenAI). This is 80-95% cheaper than hosted memory services which charge $0.15-$0.50 per call.

How long does setup take?

Approximately 15 minutes. Clone the repository, configure your .env file with API keys, run docker-compose up, and configure ElevenLabs webhooks. Our comprehensive documentation guides you through each step.

What LLM providers are supported?

Currently supports OpenAI (GPT-4, GPT-3.5) and Anthropic (Claude). You can easily extend to other providers by modifying the LLM service module. We're planning to add support for Groq, Cohere, and local models in future releases.

Is my data secure?

Yes! ELAAOMS is self-hosted on your infrastructure, so your data never leaves your servers. All ElevenLabs webhooks are authenticated using HMAC-SHA256 signatures. The system uses industry-standard security practices for data storage and transmission.

Can I use it with multiple agents?

Absolutely! ELAAOMS is designed for multi-agent environments. It can share high-importance memories across different agents, creating a unified customer experience. Each agent can also have its own private memories.

What kind of memories are extracted?

The LLM extracts five types of memories: Factual (order numbers, addresses), Preference (likes, dislikes), Issue (problems, complaints), Emotional (sentiment, mood), and Relational (names, relationships). Each memory is scored for importance.

Can I contribute to the project?

Yes! ELAAOMS is open source and welcomes contributions. Check out our CONTRIBUTING.md guide on GitHub. We're looking for help with features, documentation, testing, and integrations. Join our community Discord for discussions.
